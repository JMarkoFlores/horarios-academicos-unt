# ENTREGA P2 - Validaciones, Mapa del Campus y Análisis de Carga

## 1. Servicio de Validaciones Globales

### 1.1 Validaciones Implementadas

El sistema de validaciones globales está implementado en `ValidacionesService` y `ValidadorHorarioService`, proporcionando las siguientes validaciones:

#### 1.1.1 Cruces de Horario
- **Cruce de Docente**: Verifica que un docente no tenga otro horario en el mismo slot
- **Cruce de Ambiente**: Verifica que un ambiente no esté ocupado en el mismo slot
- **Cruce de Grupo**: Verifica que un grupo no tenga otro horario en el mismo slot

#### 1.1.2 Disponibilidad Declarada
- Verifica que el docente haya declarado disponibilidad para el día y horario solicitado
- Utiliza la entidad `DisponibilidadDocente` para validar

#### 1.1.3 Franja Institucional
- Verifica que el horario esté dentro de la franja institucional configurada
- Utiliza los turnos configurados en el sistema

#### 1.1.4 Límites de Carga
- **Máximo de horas por día**: Verifica que el docente no supere el máximo de horas permitidas en un día
- **Carga por modalidad**: Valida la carga horaria según la modalidad del docente (DEDICACION_EXCLUSIVA, TIEMPO_COMPLETO_40, etc.)
- Utiliza la entidad `ParametrosCarga` para definir los límites

#### 1.1.5 Otras Validaciones
- **Día no laborable**: Verifica que la fecha no sea un día no laborable
- **Capacidad del ambiente**: Verifica que el ambiente tenga capacidad suficiente para el grupo
- **Descanso mínimo**: Verifica que el docente tenga el descanso mínimo entre clases
- **Franja institucional**: Verifica que el horario esté dentro de la franja permitida

### 1.2 Endpoints de Validación

| Método | Endpoint | Descripción |
|--------|----------|-------------|
| POST | /horarios/validar-slot | Valida un slot de horario específico |
| GET | /horarios/conflictos/:periodo | Obtiene conflictos del período |

### 1.3 Servicios Involucrados

- **ValidacionesService**: Servicio principal de validaciones globales
- **ValidadorHorarioService**: Servicio que orquesta las validaciones de slots
- **DeteccionConflictosService**: Servicio para detectar y tipificar conflictos

## 2. Mapa Interactivo del Campus

### 2.1 Entidad Ambiente - Campos de Ubicación

La entidad `Ambiente` incluye los siguientes campos para el mapa:

```typescript
@Column({ nullable: true })
piso: number;

@Column({ nullable: true, length: 50 })
pabellon: string;

@Column({ nullable: true, length: 100 })
edificio: string;

@Column({ type: "float", nullable: true, name: "coord_x" })
coordX: number;

@Column({ type: "float", nullable: true, name: "coord_y" })
coordY: number;

@Column({ nullable: true, length: 100 })
sede: string;

@Column({ type: "text", nullable: true })
equipamiento: string;
```

### 2.2 Endpoints del Mapa

| Método | Endpoint | Descripción |
|--------|----------|-------------|
| GET | /ambientes/mapa | Lista ambientes activos con coordenadas para mapa |
| GET | /ambientes/distancia?origenId=&destinoId= | Calcula distancia entre dos ambientes |
| GET | /ambientes/alertas-traslado?docenteId=&periodoId= | Obtiene alertas de traslado entre horarios consecutivos |

### 2.3 Respuesta del Endpoint /ambientes/mapa

```json
{
  "data": [
    {
      "id": 1,
      "nombre": "Aula 101",
      "coordX": 100.5,
      "coordY": 200.3,
      "edificio": "Edificio Principal",
      "capacidad": 40
    }
  ],
  "message": "Ambientes para mapa obtenidos correctamente"
}
```

### 2.4 Cálculo de Distancia

El sistema calcula la distancia entre ambientes usando la fórmula euclidiana:

```
distancia = sqrt((coordX_destino - coordX_origen)^2 + (coordY_destino - coordY_origen)^2)
```

#### 2.4.1 Alertas de Traslado

El sistema genera alertas cuando:
- Los ambientes están en edificios diferentes
- La distancia entre ambientes supera el umbral configurado (ALERTA_DISTANCIA_MAX)
- El tiempo entre clases consecutivas es menor a 30 minutos

#### 2.4.2 Respuesta del Endpoint /ambientes/distancia

```json
{
  "data": {
    "distanciaUnidades": 150.5,
    "mismoEdificio": false,
    "alertaTraslado": true
  },
  "message": "Distancia entre ambientes calculada correctamente"
}
```

### 2.5 Frontend - Módulo Mapa Campus

El módulo `mapa-campus` en el frontend proporciona:
- Vista interactiva del campus con coordenadas
- Visualización de ambientes en el mapa
- Cálculo de distancias entre ambientes
- Alertas visuales para traslados largos

### 3.2 Carga Desequilibrada

#### 3.2.1 Endpoint

| Método | Endpoint | Descripción |
|--------|----------|-------------|
| GET | /docentes/carga-desequilibrada?periodo= | Lista docentes con carga horaria desequilibrada |

#### 3.2.2 Cálculo de Desequilibrio

El sistema calcula el desequilibrio usando la desviación estándar de la carga por día:

```
desequilibrio = sqrt(sum((horas_dia - promedio)^2) / dias_con_carga)
```

Se considera desequilibrado cuando el desequilibrio > 4 horas.

#### 3.2.3 Respuesta

```json
{
  "data": [
    {
      "docenteId": 1,
      "nombre": "Juan Pérez",
      "distribucion": {
        "lunes": 8,
        "martes": 0,
        "miercoles": 8,
        "jueves": 0,
        "viernes": 4,
        "sabado": 0,
        "totalHoras": 20,
        "promedioHorasPorDia": 6.67
      },
      "desequilibrio": 4.5
    }
  ],
  "message": "Carga desequilibrada obtenida correctamente"
}
```

### 3.3 Validación de Carga por Modalidad

#### 3.3.1 Parámetros de Carga

La entidad `ParametrosCarga` define los límites de carga según:
- Modalidad (DEDICACION_EXCLUSIVA, TIEMPO_COMPLETO_40, etc.)
- Tipo de docente (ORDINARIO, CONTRATADO, etc.)
- Categoría (PRINCIPAL, ASOCIADO, AUXILIAR)

#### 3.3.2 Valores por Defecto

| Modalidad | Horas Mínima Semanal |
|-----------|---------------------|
| DEDICACION_EXCLUSIVA | 40 |
| TIEMPO_COMPLETO_40 | 40 |
| TIEMPO_PARCIAL_20 | 20 |
| TIEMPO_PARCIAL_12 | 12 |
| TIEMPO_PARCIAL_10 | 10 |
| TIEMPO_PARCIAL_8 | 8 |

### 3.4 Frontend - Módulo Análisis Carga

El módulo `analisis-carga` en el frontend proporciona:
- Gráfico de distribución de carga por día de semana (usando Chart.js)
- Lista de docentes con carga desequilibrada
- Heatmap de ocupación por hora/día
- Indicador de cumplimiento de carga vs mínimo normativo

#### 3.4.1 Indicador de Cumplimiento

El indicador de cumplimiento muestra:
- **Verde (check_circle)**: Cumplimiento >= 100%
- **Azul (trending_up)**: Cumplimiento >= 80%
- **Naranja (warning)**: Cumplimiento >= 50%
- **Rojo (error)**: Cumplimiento < 50%

## 7. Coordenadas de Aulas (Escuela de Postgrado UNT)

Para configurar las coordenadas de las aulas de la Escuela de Postgrado UNT, se deben actualizar los campos `coordX`, `coordY`, `edificio`, `piso` y `pabellon` en la entidad Ambiente.

### 7.1 Ejemplo de Configuración

```sql
UPDATE ambiente SET 
  coord_x = 100.5,
  coord_y = 200.3,
  edificio = 'Escuela de Postgrado',
  piso = 1,
  pabellon = 'Pabellón A',
  sede = 'Escuela de Postgrado UNT'
WHERE codigo = 'AULA-101';
```

Para configurar las coordenadas de los laboratorios del segundo piso de la Oficina de Registro Técnico, se deben actualizar los campos correspondientes.
